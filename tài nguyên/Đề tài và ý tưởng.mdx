# 1. Đề tài

> **Thiết kế và triển khai hệ thống IoT giám sát đất và không khí, điều khiển tưới tiêu theo thời gian thực**

---

# 2. Mục tiêu hệ thống

Xây dựng hệ thống IoT thông minh giúp tối ưu hóa việc giám sát và chăm sóc cây trồng:

- **Giám sát thời gian thực:**
  - Độ ẩm đất (thông qua Potentiometer mô phỏng)
  - Nhiệt độ và độ ẩm không khí (Cảm biến DHT22)
  - Cường độ ánh sáng (Cảm biến báo mức sáng BH1750)
- **Điều khiển và Tự động hóa:**
  - Điều khiển máy bơm nước qua module Relay.
  - Tích hợp chế độ tưới tự động (Auto Mode) hoạt động độc lập ngay trên thiết bị vi điều khiển ESP32 nhằm đảm bảo tính phản hồi tức thời.
- **Trực quan hóa dữ liệu:**
  - Dashboard Node-RED cung cấp giao diện hiển thị các chỉ số hiện tại và biểu đồ tương tác đồ họa.
- **Cảnh báo thông minh:**
  - Cảnh báo nhắc nhở dạng Toast UI khi các chỉ số vượt ngưỡng an toàn (Nét bật: Nhiệt độ cao, Đất khô, Thiếu sáng).
  - Còi báo động (Buzzer) phát âm phím vật lý nhằm cảnh báo sự kiện hệ thống rơi vào mức độ nguy hiểm (chẳng hạn thiếu nước trầm trọng).

---

# 3. Kiến trúc hệ thống

Hệ thống hoạt động dựa trên mô hình hệ thống IoT 3 lớp tiêu chuẩn:

### 1. Device Layer (Lớp thiết bị)
- Phụ trách thu thập dữ liệu và vận hành phần cứng:
  - Vi điều khiển trung tâm: **ESP32** (Sử dụng Wokwi)
  - Cảm biến môi trường: **DHT22** (Nhiệt độ, Độ ẩm không khí), **BH1750** (Cường độ ánh sáng giao tiếp I2C).
  - Cảm biến đất: **Potentiometer** giả lập đo đạc thông qua ADC.
  - Cơ cấu chấp hành và cảnh báo: **Relay module** điều khiển bơm nước và loa **Buzzer** cảnh báo.

### 2. Network Layer (Lớp truyền tải mạng)
- Cơ chế liên lạc đa điểm **MQTT** qua giao thức bảo mật mã hoá **TLS/SSL** (port 8883) kết nối tới Broker **HiveMQ Cloud**.
  - Đầu phát dữ liệu (Publish): `iot/farm/env`
  - Đăng ký nhận lệnh điều khiển máy (Subscribe): `iot/farm/control`
  - Đăng ký nhận lệnh thay đổi cấu hình (Subscribe): `iot/farm/config`

### 3. Application Layer (Lớp ứng dụng quản trị)
- Dạng luồng xử lý chính: **Node-RED**. Hứng luồng dữ liệu (messages) từ Broker, rẽ nhánh đưa ra các chỉ báo/giao diện người dùng trực quan trên **Node-RED Dashboard** và phát lệnh xử lý logic xuống ESP32.

# 4. Sơ đồ Phần cứng (Wokwi)

| Linh kiện | Chân kết nối ESP32 | Vai trò |
|-----------|--------------------|---------|
| **ESP32 DevKit C v4** | — | Vi điều khiển xử lý trung tâm. |
| **DHT22** | GPIO 4 | Đo nhiệt độ và độ ẩm không khí. |
| **Potentiometer** | GPIO 34 (ADC) | Biến trở giả lập độ ẩm đất. |
| **BH1750** | GPIO 21 (SDA), 22 (SCL) | Cảm biến cường độ sáng giao tiếp I2C. |
| **Relay Module** | GPIO 26 | Đóng ngắt máy bơm nước (HIGH/LOW). |
| **Buzzer** | GPIO 27 | Chuông phát tín hiệu báo động khi phát cảnh báo tại khu vực trồng. |

# 5. Cấu trúc bản tin MQTT

Các luồng dữ liệu trao đổi bằng định dạng JSON nhằm tối ưu việc phân tích thông số hệ thống.

**1. Gửi dữ liệu cảm biến (ESP32 -> Node-RED)**
Topic: `iot/farm/env`
```json
{
  "device": "AgriSense-01",
  "temperature": 32.5,
  "humidity": 65.2,
  "soilMoisture": 25.0,
  "lightLux": 1200.5,
  "relayStatus": "OFF",
  "timestamp": 123456789
}
```

**2. Nhận lệnh điều khiển bơm (Node-RED -> ESP32)**
Topic: `iot/farm/control`
```json
// Bật bơm trong một mốc thời gian tự động tắt (ví dụ 5 giây)
{
  "pump": true,
  "duration": 5
}
```

**3. Lệnh cấu hình hệ thống, tưới tự động (Node-RED -> ESP32)**
Topic: `iot/farm/config`
```json
// Chuyển đổi trạng thái chế độ tưới thông minh (Auto Mode)
{
  "auto": true
}
```

# 6. Logic vận hành & Ngưỡng cảnh báo

Hệ thống có logic vận hành tự duy trì và cảnh báo chủ động tại vùng trồng:

| Tình trạng biến động | Ngưỡng phát hiện | Thao tác của hệ thống |
|----------------------|------------------|-----------------------|
| **Đất hơi khô** | Độ ẩm < 30% | Nhắc nhở Toast trên Dashboard "Đất khô — cần tưới". Kích hoạt ESP32 rung còi bíp 2 lần (dùng timer). Có thể kích hoạt bơm tự động nếu Mode Auto đang bật. |
| **Đất khô nguy khốn**| Độ ẩm < 20% | Hiện cảnh báo ruy-băng đỏ trên màn hình thao tác. ESP32 sẽ hú còi kịch liệt (bíp 5 lần) báo độ nghiêm trọng. |
| **Nhiệt độ nóng bức**| Nhiệt độ > 35°C | Cửa sổ thông báo (Toast UI) khuyến nghị áp dụng các biện pháp làm mát phòng trồng. |
| **Ngưỡng thiếu sáng** | Ánh sáng < 1000 lux | Báo động thiếu sáng. Hiển thị UI Gauge báo đỏ để báo nguy cơ cho quá trình quang hợp của cây. |

# 7. Giao diện Dashboard Node-RED

Gồm 4 cụm điều hướng chính, tối đa hóa trải nghiệm điều khiển:

1. **Vùng Đất & Hệ thống Tưới:**
   - Gauge Donut màu sắc tương hỗ cho độ ẩm đất, biểu đồ lịch sử 1 giờ dạng Line.
   - Trạng thái bơm tức thời (Bật/Tắt đồng bộ).
   - Bảng mạch cảnh báo nguy cấp.
2. **Vùng Không Khí Sinh Cảnh:**
   - Khối số học sinh động hiện Nhiệt độ trực quan (đơn vị °C) và Độ Ẩm Không khí.
   - Đồ thị quá trình dao động nhiệt ẩm.
3. **Vùng Đo Lường Ánh Sáng:**
   - Gauge Donut biến màu ranh giới (xanh, vàng, đỏ) thông qua dải lux tự nhiên. Hiển thị lớn chỉ số lux với hệ thống cảnh báo đa diện.
4. **Bảng Trình Điều Khiển Thiết Bị:**
   - Cung cấp nút thao tác chủ động điều lệnh để ép cấp hoặc ngừng cấp nguồn nước (Manual Pump). Phím switch chuyên đổi kích hoạt Logic (Auto/Manual). Mọi thao tác này sẽ sinh ra gói tin JSON và nhả vào MQTT Topic `control` tương thích.
